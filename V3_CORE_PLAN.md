# V3.0 Core 第一階段開發清單

## 版本目標

- 完成 V3.0 Core 基礎功能，建立穩定、安全、可擴充的核心架構。
- 讓戰鬥系統從舊版升級為完整的 Battle System 3.0。
- 補齊中後期內容需求，包含世界 Boss、精英怪、怪物圖鑑與戰鬥表現。

## 階段一總覽

- 專案基礎重構
- 穩定性與安全性補強
- 戰鬥系統 3.0
- 內容擴充
- 戰鬥表現優化

## 一、專案基礎重構

- [ ] 重構專案架構
- [ ] 模組重新分層
- [ ] 整理共用工具函式
- [ ] 統一資料流與事件流程
- [ ] 降低模組耦合度
- [ ] 補齊必要註解與文件

### 驗收標準

- 主要模組職責清楚。
- 戰鬥、資料、UI、儲存可分離維護。
- 新功能可在不大改舊程式的前提下擴充。

## 二、穩定性與安全性補強

- [ ] 修復已知 Bug
- [ ] 深度漏洞檢查
- [ ] 新增 `security.js`
- [ ] JSON 格式驗證
- [ ] 自動存檔
- [ ] 更新檢查

### 驗收標準

- 已知問題有修正紀錄。
- 輸入資料有驗證與防呆。
- 存檔資料損壞時可攔截或回復。
- 版本更新資訊可正確檢查與提示。

## 三、戰鬥系統 3.0

- [ ] 普攻系統
- [ ] 技能系統
- [ ] 必殺技系統
- [ ] 暴擊判定
- [ ] 閃避判定
- [ ] 格擋判定
- [ ] 元素相剋系統
- [ ] 狀態效果系統
- [ ] 技能冷卻（CD）

### 狀態效果範圍

- [ ] 中毒
- [ ] 燃燒
- [ ] 冰凍
- [ ] 其他可擴充異常狀態

### 驗收標準

- 每回合或每次行動流程可正常結算。
- 傷害、命中、暴擊、閃避、格擋邏輯正確。
- 技能與必殺技可獨立設定效果。
- 狀態效果可套用、持續、移除。
- CD 可正確進入、遞減、重置。
- 元素相剋會實際影響戰鬥結果。

## 四、內容擴充

- [ ] 世界 Boss
- [ ] 精英怪
- [ ] 怪物圖鑑

### 驗收標準

- Boss 擁有獨立定位與強化機制。
- 精英怪與普通怪有明確差異。
- 圖鑑可記錄怪物資訊、屬性與遭遇狀態。

## 五、戰鬥表現優化

- [ ] 傷害飄字
- [ ] 技能特效

### 驗收標準

- 攻擊與技能觸發時有即時回饋。
- 飄字不影響戰鬥判定。
- 特效可依技能或屬性切換。
- 表現層與邏輯層分離。

## 建議實作順序

1. 專案架構重構
2. `security.js`、JSON 驗證、自動存檔
3. 已知 Bug 修復、漏洞檢查、更新檢查
4. 戰鬥系統 3.0
5. 世界 Boss、精英怪、怪物圖鑑
6. 傷害飄字、技能特效

## 里程碑版本建議

- `v3.0.1`：架構重構、Bug 修復、安全與資料驗證
- `v3.0.2`：戰鬥系統 3.0 核心完成
- `v3.0.3`：Boss、精英怪、圖鑑完成
- `v3.0.4`：飄字、特效、整體調校完成

## 建議任務標籤

- `core`

## 一句話版本定位

V3.0 Core 是以「穩定性 + 安全性 + 戰鬥系統全面升級」為核心的第一階段主版本。
